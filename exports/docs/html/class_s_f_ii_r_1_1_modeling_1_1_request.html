Метод для генерации простейшего потока заявок arrivalLambda - среднее количество событий за единицу времени, то есть среднее время ожидания нового события равно 1 / arrivalLambda .

Если <paramref name="arrivalLambda"/> = <see langword="0.5"/> -> среднее время ожидания новой заявки = <b>1 / <see langword="0.5"/> = 2.0</b> ед. времени <br/>
Если <paramref name="arrivalLambda"/> = <see langword="2.0"/> -> среднее время ожидания новой заявки = <b>1 / <see langword="2.0"/> = 0.5</b> ед. времени

memLambda - это параметр интенсивности экспоненциального распределения, которое определяет, сколько памяти потребуется для создания программного процесса из заявки

Если <paramref name="memLambda"/> = <see langword="0.5"/> -> средняя память = <b>1 / <see langword="0.5"/> = 2.0</b> единицы <br/>
Если <paramref name="memLambda"/> = <see langword="2.0"/> -> средняя память = <b>1 / <see langword="2.0"/> = 0.5</b> единицы

cpuLambda - Среднее количество событий (математическое ожидание) и дисперсия

Если <paramref name="cpuLambda"/> = <see langword="5"/> -> среднее время выполнения = <b><see langword="5"/></b> квантов <br/>
Если <paramref name="cpuLambda"/> = <see langword="20"/> -> среднее время выполнения = <b><see langword="20"/></b> квантов
